What to Self-Host on a Raspberry Pi 5 (8GB)

June 1, 2026 · 6 min read
Self-Hosting Raspberry Pi Homelab Linux

The Raspberry Pi 5 with 8GB RAM is surprisingly capable as a self-hosting box. But with limited RAM and an ARM processor, you need to be strategic about what you run. Here's a practical guide based on real experience — what works, what doesn't, and what's actually worth your time.

The Top Picks

Nextcloud — Your Own Cloud

File sync, calendar, contacts, photos — all replacing Google/Apple services. This is the single most impactful self-hosted service you can run.

✓ Replaces Google Drive, Google Photos, Calendar, Contacts

✓ Mobile apps auto-sync photos and files

✓ Share files with links like Dropbox

✗ Can be RAM-hungry with many apps enabled

Tip: Use it with Tailscale for secure remote access without exposing ports. Works beautifully.

Pi-hole / AdGuard Home — Network-Wide Ad Blocking

Every device on your network gets ad blocking for free. Set it as your DNS server and watch the ads vanish.

✓ Zero config on client devices

✓ Instantly improves daily browsing

✓ Super light on resources (~50MB RAM)

✗ Some apps break if they rely on ad domains (easy whitelist fix)

Immich — Google Photos Replacement

Auto-backup photos from your phone, search by face/object, share albums. It's genuinely beautiful software.

✓ Facial recognition, map view, shared albums

✓ Mobile app auto-upload

✗ Machine learning features need some RAM

✗ If you already have Nextcloud, consider Nextcloud Photos instead to save RAM

Uptime Kuma — Service Monitor

"Here's my status page with 15 services, all green." That alone is worth it. Also pings you when things break.

✓ Beautiful status pages

✓ Push, HTTP, DNS, ping monitoring

✓ Telegram/Discord/email alerts

✗ Not really a con, it's just great

Coding and AI

Ollama + Open WebUI — Local LLM Chat

Run small models (Llama 3.2 3B, Qwen 2.5 7B) locally on your Pi. Yes, ARM works. No, it won't be fast — but it's YOUR AI.

✓ Privacy: no data leaves your network

✓ Great for "I built my own AI" resume factor

✗ 8GB shared RAM limits you to ~3B models realistically

✗ Speed: expect 3-8 tok/s on small models

Caddy Reverse Proxy

Give every service its own subdomain with automatic HTTPS. One clean config file replaces Nginx complexity.

✓ Auto-HTTPS with Let's Encrypt

✓ Simpler config than Nginx

✓ Essential if you're running multiple services

Quality of Life

Audiobookshelf — Audiobooks and Podcasts

If you're already self-hosting media, this completes the suite. Clean, fast, works great.

Paperless-ngx — Document Management

Scan, OCR, search. Find any document in seconds. Your future self will thank you.

Navidrome — Music Streaming

Your own Spotify. Stream your music collection from anywhere. Ultra-light on resources.

The Ideal 8GB Pi 5 Stack

Nextcloud (files/calendar/contacts)
+ Pi-hole (DNS ad blocking)
+ Caddy (reverse proxy + HTTPS)
+ Uptime Kuma (monitoring)
+ Immich OR Nextcloud Photos (pick one to save RAM)
+ Audiobookshelf (podcasts/audiobooks)

Optional if RAM allows:
+ Ollama + Open WebUI (local LLM)
+ Paperless-ngx (documents)

What NOT to Run on 8GB

  • Video transcoding (Plex/Jellyfin with transcode): The Pi 5 can serve pre-transcoded files fine, but live transcoding will choke.
  • CI/CD runners (Gitea Actions, Drone): Compiling on ARM + 8GB RAM = pain if you have any real dependencies.
  • Large LLMs (14B+): 8GB shared RAM means the model + OS + other services = OOM. Stick to 3B-7B quantized models.
  • Multiple databases: Each service wants PostgreSQL? Consolidate or use SQLite where possible.

RAM-Saving Tips

  • Use SQLite instead of PostgreSQL where the service supports it
  • Limit PHP-FPM workers in Nextcloud (default spawns too many)
  • Use Docker with memory limits: deploy.resources.limits.memory: 512M
  • Add swap: sudo fallocate -l 2G /swapfile (SSD/boot drive only, not SD card)
  • Monitor with htop and docker stats to catch RAM hogs early
