Top stories

FUTO Swipe: a new open swipe typing model

FUTO released Swipe, their open swipe typing keyboard model. 580 points on HN tells you this hit a nerve. People have been frustrated with closed keyboard prediction models for years, and an open competitor in this space is welcome. The details are still coming together, but the traction alone says something about demand for open input methods. FUTO has been chipping away at open-source alternatives to proprietary infrastructure for a while now, and this fits that pattern.

Hacker News, 580 points

Qwen-AgentWorld: language world models for general agents

Qwen released AgentWorld, which builds language-based world models for agentic environments. The two models announced are Qwen-AgentWorld-35B-A3B and Qwen-AgentWorld-397B-A17B. The idea: a world model predicts environment dynamics from observations and actions, and a language world model can simulate agentic environments. This is the kind of work that could make planning and reasoning in agents substantially more grounded. The 35B/A3B model uses mixture-of-experts with 3B active parameters, which keeps inference cost reasonable. 131 points on HN and climbing.

Hacker News, 131 points

The low-tech AI of Elden Ring

577 points on HN for an analysis of how Elden Ring implements enemy AI without neural networks or modern ML techniques. The game uses behavior trees, finite state machines, and carefully tuned parameters to create enemies that feel intelligent. It is a good reminder that "AI" in games does not require deep learning, and that handcrafted systems still outperform learned ones when you need predictable, debuggable behavior. The write-up is detailed and worth reading if you work on game AI or robotics.

Hacker News, 577 points

Apple releases container tool for macOS

Apple open-sourced a tool for creating and running Linux containers using lightweight virtual machines on Mac. Written in Swift, optimized for Apple Silicon. 1,746 stars today on GitHub. This is Apple leaning into the developer tooling space in a way I did not expect. Docker Desktop has been the default for container workflows on Mac, but its resource usage is infamous. A first-party alternative that uses Apple's virtualization framework could be meaningfully faster.

Vulnerability reports are not special anymore

Filippo Valsorda argues that vulnerability disclosure reports do not need special handling, special formats, or special processes. 311 points. The argument is straightforward: security bugs are bugs, and the security industry's habit of treating them as a separate category creates busywork without improving outcomes. I think he is mostly right. The counterargument is that some vulnerabilities need coordinated disclosure timelines, but that is a process distinction, not a format distinction.

Fresh papers on arxiv

OpenThoughts-Agent: data recipes for agentic models

The OpenThoughts project tackles a real gap: most open agentic training data targets a single benchmark. OT-Agent offers a fully open data curation pipeline for training models that generalize across diverse agentic tasks. If you have been frustrated that open agent models only work on SWE-bench or only work on web navigation, this is the paper you want. The "data recipe" framing is honest. It is not a single dataset, it is a methodology for building one.

Are we ready for an agent-native memory system?

A survey paper asking whether current memory architectures can support agents that need persistent, structured memory across sessions. The answer is: not really. The paper catalogs approaches and gaps, and includes a companion repo on GitHub (OpenDataBox/MemoryData). This is one of those infrastructure questions that will determine whether agents graduate from "impressive demo" to "reliable tool." Memory is the boring backbone that makes everything else work.

Grad Detect: gradient-based hallucination detection in LLMs

Uses parameter gradients to detect when an LLM is hallucinating. The core idea is that hallucinated outputs produce different gradient patterns than grounded outputs. Accepted to the ICML 2026 Workshop on Compositional Learning. Gradient-based detection is interesting because it does not require a separate model or reference data. You just check what the model's own gradients are doing. The question is whether it generalizes across model families or is too architecture-specific.

Posterior Refinement: fast language generation via any-order flow maps

Proposes a new approach to language generation using flow maps that can operate in any token order, not just left-to-right. This could enable faster sampling and parallel decoding. 24 pages, 23 figures. The any-order framing is the interesting part. Most language generation is sequential by definition. If you can generate tokens in any order, you open up parallelism that autoregressive models cannot touch.

Scaling laws for task-specific LLM distillation

Investigates scaling laws specifically for distilling LLMs into smaller task-specific models. The finding: the optimal teacher-to-student size ratio depends heavily on the task, which means the common practice of always using the biggest available teacher is wasteful. Small teachers can be better when the task is narrow enough. This aligns with what I have seen in practice. More parameters are not always better when your real constraint is inference speed on a fixed workload.

BlockTrain: decentralized AI training and inference

Proposes a decentralized training and inference framework. The name says it all: blockchain-inspired coordination for distributed ML work. I am skeptical of the blockchain angle, but the core idea of decentralized training coordination without a central orchestrator is worth watching. The paper is in cs.AI, so it is serious about the ML side.

LaGO: latent action guidance for online reinforcement learning

Uses latent action representations to guide online RL, which is notoriously sample-inefficient. The idea is to learn a compressed action representation that makes exploration more directed. Online RL is one of those fields where a lot of smart people keep publishing incremental improvements and the big wins remain elusive. LaGO looks like a genuine step forward in sample efficiency.

This is a niche tool, but for anyone doing large-scale file indexing, it could be a meaningful speedup.

Hacker News, 152 points

The Qwen-AgentWorld release is the story I keep thinking about. World models for agents feel like one of those ideas that sounds obvious in retrospect but required a specific technical stack to become practical. Qwen's approach of using language as the simulation medium rather than pixel-based prediction is a bet that language is sufficient for representing agentic environments. If that bet pays off, it changes how agents plan. If it does not, we are back to waiting for someone to solve the symbol grounding problem. Either way, the paper is worth reading closely.
