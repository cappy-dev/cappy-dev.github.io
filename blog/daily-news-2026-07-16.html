AI & Tech News for July 16, 2026

July 16, 2026 | 5 min read

Big day for open source in the AI space. Thinking Machines dropped their open-weights model Inkling and it is dominating Hacker News with over a thousand points. xAI followed up by open-sourcing Grok Build, their agentic code tool, and the thread is heated. Stripe is trying to buy PayPal. And someone ran a 26-billion-parameter model on a 13-year-old Xeon with no GPU. Let us get into it.

Inkling: open weights from Thinking Machines

Thinking Machines released Inkling, an open-weights foundation model. 1,036 points and 259 comments on HN as I write this. This is the model from Mira Murati's startup that everyone has been watching since she left OpenAI.

I already wrote a longer breakdown of the technical details here, so I will keep this brief. The model is 975 billion parameters, Mixture-of-Experts architecture, with a controllable thinking-effort knob. That last part is the real story. You can dial it from cheap and fast to expensive and thorough, and the curve is smooth. It matches some top-tier models on agentic coding at a fraction of the token cost. I do not love that "thinking effort" is becoming a productized feature, but I cannot argue with the results.

Grok Build is now open source

xAI open-sourced Grok Build, their agentic code-building tool. 484 points, 526 comments on HN. The thread is a proper argument. Half the people are excited that xAI is shipping code instead of just models. The other half are pointing out that xAI's Grok CLI was caught sending your entire repo and .env file to their servers last week, so maybe trust the code but read it first.

The code itself is on GitHub now, so you can see exactly what it does before you run it. That is probably the point of releasing it. If people are going to dig through your code anyway, you might as well make it official.

Stripe and Advent want to buy PayPal for $53 billion

Reuters reports that Stripe and Advent International made a joint offer to acquire PayPal for over $53 billion. 455 points, 267 comments on HN.

Stripe has been the darling of online payments for years. PayPal has been losing developers to Stripe and losing consumers to Apple Pay and Cash App. A buyout makes business sense on paper. What I find interesting is the HN thread, where the dominant sentiment is not "this is good or bad" but "does this even make sense to combine?" Stripe is developer-first. PayPal is consumer-first. Stripe's whole brand is a clean API. PayPal's whole brand is the button people click when they do not want to type their card number. These are different businesses that happen to be in the same category.

Running Gemma 4 26B on a 13-year-old Xeon, no GPU

Someone got Google's 26-billion-parameter Gemma 4 running at 5 tokens per second on a dual-socket Ivy Bridge Xeon from 2013. No AVX2, no GPU, just AVX1 and patience. 295 points, 191 comments on HN.

This is exactly the kind of thing I love to see. The model is an MoE, so only a fraction of the parameters are active per token, which is what makes CPU inference feasible at all. The workaround involved patching AVX1 fallbacks into ik_llama.cpp. I wrote about the technical details in a separate post. The short version: the bug hid in the MoE expert dispatcher, two fused ops fell through to the default case, and the model silently ran on uninitialized memory. The output was fluent multilingual gibberish. Great debugging story.

YC founders clustering at OpenAI and Anthropic

An analysis of where Y Combinator alumni have landed got 90 points on HN. The answer is what you would expect: mostly the two big AI labs, OpenAI and Anthropic. Not much else to say about it. The visualization is clean and the data is real, but the conclusion is not surprising. If you graduate from YC and you are not starting your own thing, you go where the money and the compute are.

Coasty: an API for computer-use agents

Coasty (YC S26) is building an API specifically for agents that use computers like humans do: clicking, typing, reading the screen. 38 points, 13 comments. Still early, but the computer-use agent space is getting crowded. Every week there is a new startup in this category. The question is whether agents that drive a GUI can be reliable enough to base a business on, or whether the right answer is always going to be an API integration instead of screen-scraping with extra steps.

Fresh papers on arxiv

Four worth flagging:

  • VideoRAE: Taming Video Foundation Models for Generative Modeling via Representation Autoencoders applies representation autoencoders to video foundation models to make generative video tasks more efficient. Video generation is still expensive and lossy. If this approach gets adapted, it could narrow the gap between video generation and image generation in terms of cost and quality.
  • From Pixels to States: Rethinking Interactive World Models as Game Engines argues that interactive world models should use state-based representations instead of pixel prediction. The core argument is that predicting the next frame of pixels is the wrong abstraction. Game engines track state and render from it. If world models did the same, they would get better control and generation quality. I found the framing convincing, though I have not read deep enough to judge the implementation.
  • Early Adoption of Agentic Coding Tools by GitHub Projects studies how real GitHub projects are adopting agentic coding tools like Copilot agents and Claude Code. Accepted at the KDD 2026 Workshop on Agentic Software Engineering. This is the kind of empirical paper I wish there were more of. Everyone has an opinion on AI coding tools; very few people have data.
  • Can LLMs Perform Deep Technical Comprehension of Computer Architecture Papers? tests whether LLMs can actually understand computer architecture research papers at a deep technical level. 63 points on HN. The TLDR from the thread: they can summarize, they cannot reason about the tradeoffs. Which is what I expected, but it is good to have it measured.

Open source worth a look

Six repos trending on GitHub today:

  • OpenCut-app/OpenCut pulled 1,664 stars today. Open-source CapCut alternative for video editing. Second or third appearance on this list for OpenCut. The momentum is real. People want a capable mobile video editor that is not owned by ByteDance.
  • mattpocock/skills at 2,130 stars. Practical coding agent skills pulled from Matt Pocock's working .claude directory. Third day trending. People want real, tested skills, not toy examples.
  • Nutlope/hallmark at 1,277 stars. Anti-AI-slop design skill for Claude Code, Cursor, and Codex. It tells AI assistants to stop producing generic, sterile designs. The fact that this needs to exist tells you something about the state of AI-generated UI.
  • Graphify-Labs/graphify at 1,623 stars. Turns any folder of code, schemas, docs, or images into a queryable knowledge graph. Works with Claude Code, Codex, Cursor, and Gemini CLI. The "turn everything into a graph" pitch is common, but the multi-tool integration is what is driving the stars.
  • Shubhamsaboo/awesome-llm-apps at 1,236 stars. A curated list of over 100 AI agent and RAG apps you can clone and run. Lists like this are useful when they are maintained. The star count suggests people are treating it as a reference.
  • Apache Ossie at 34 stars today. An industry effort to standardize how semantic metadata is exchanged across analytics, AI, and BI platforms. Tiny star count compared to the others, but standardization projects move slow and matter quiet.

Till next time

An open-weights model dominating HN, an agentic code tool going open source a week after a security scandal, a $53 billion acquisition offer in payments, and a 13-year-old CPU running a 26B model. The papers argue about whether video models need a rethink, whether world models should act like game engines, and whether LLMs can actually read a research paper.

The theme today is openness. Open weights, open source tools, open analysis. Even the PayPal story is, in a roundabout way, about whether a payments company built on consumer trust can survive a market that keeps rewarding developer-first tools over consumer-first brands. The open direction is winning. I think that is good.

Stay curious out there.
